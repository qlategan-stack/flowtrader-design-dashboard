System Overview
What FlowTrader is and how it works at a glance
🎯

Strategy

CLAUDE.md

Mean reversion — buys when an asset is statistically oversold relative to its short-term average, exits when price returns to the mean.

  • Indicators: Bollinger Bands, RSI, VWAP, ADX
  • Signal gate: 3 of 6 signals must fire
  • Target exit: 20-day moving average
🌍

Universe

config.yaml

Trades US equities and ETFs via Alpaca. Crypto support via CCXT (Binance) is in the config but not the primary focus.

  • Equities: NVDA, AAPL, MSFT, QQQ, SPY, AMD, GLD
  • Crypto: BTC/USDT, ETH/USDT, SOL/USDT
  • Broker: Alpaca (paper & live)
⚙️

Execution Mode

.env / secrets

Paper trading is the default and requires explicit dual-flag override to go live. Capital safety is the top priority.

  • Default: Paper trading on Alpaca sandbox
  • Live gate: Two env flags must both be set
  • Notifications: Telegram alerts on every trade

Agents
Six distinct agents handle research, decision, execution, and improvement

TradingDecisionAgent

agents/decision.py · claude-sonnet-4-6
Purpose

The brain of the system. Receives a full market snapshot (prices, indicators, news sentiment, account status) and uses the Claude claude-sonnet-4-6 model to output a structured JSON trade decision. The system prompt is loaded from CLAUDE.md, which encodes all hard rules and signal scoring definitions.

Key Methods
  • analyze_market(market_snapshot, account_snapshot) → dict
  • analyze_single_symbol(symbol_data, account) → dict
  • run_weekly_review(entries) → str
Output Format
BUY / SELL / HOLD / SKIP signal_score 0-6 entry + stop_loss + take_profit reasoning + journal_entry
⚡

OrderExecutor

agents/executor.py · Alpaca Trading API
Purpose

Validates every trade decision against hard guardrails before submitting to Alpaca. Acts as a final safety layer — even if Claude returns a BUY, the executor will reject if any guardrail fails. Defaults to paper trading; live requires two env flags.

Key Methods
  • validate_order(symbol, side, qty, entry, stop, acct_val, positions, day_pl) → (bool, str)
  • place_order(decision, account) → dict
Guardrails Enforced
Max 3 positions 2% daily loss limit 1% max position size Stop loss required Paper default
— Analyst Agents (run once daily at 17:30 EST) —

InStrategyAnalyst

agents/analyst_in.py · claude-sonnet-4-6
Purpose

Reviews the trade journal (last N days) and produces suggestions for tuning the existing mean reversion strategy. Does NOT suggest new strategies. Expert in Bollinger Band, RSI, VWAP, ADX, and ATR parameter calibration. Produces suggestions with current vs proposed rules and a proposed CLAUDE.md diff.

Key Methods
  • run(days=30) → list[str] # suggestion IDs created
  • _build_prompt(entries, claude_md) → str
  • _call_claude(prompt) → str | None
  • _parse_suggestions(raw) → list[dict]
Suggestion Categories
rsi_threshold bollinger_params adx_filter atr_stop_multiplier signal_score_threshold vwap_deviation

OutStrategyAnalyst

agents/analyst_out.py · claude-sonnet-4-6 + yfinance
Purpose

A senior market strategist who zooms out. Reads the journal plus live macro context — VIX, SPY trend, 5-sector ETF performance — and asks: does this strategy fit the current market regime? Suggests watchlist changes, new strategies to layer in, and when to pause.

Key Methods
  • run(days=30) → list[str] # suggestion IDs created
  • _fetch_macro_context() → dict # VIX, SPY, sectors
  • _build_prompt(entries, claude_md, macro) → str
  • _call_claude(prompt) → str | None
Macro Data Sources (yfinance)
^VIX (fear gauge) SPY 30d MA20 XLK Technology XLE Energy XLF Financials XLV Healthcare XLI Industrials

MarketDataFetcher

data/fetcher.py · Alpaca + Alpha Vantage
Purpose

Pulls everything Claude needs for a trade decision into a single clean JSON snapshot. Fetches price history, computes all technical indicators (RSI, Bollinger Bands, VWAP, ADX, ATR), pulls news sentiment from Alpha Vantage, and assembles a ranked watchlist by signal score.

Key Methods
  • get_account_snapshot() → dict # balance, positions, P&L
  • build_market_snapshot(watchlist) → dict # ranked by score
  • get_symbol_indicators(symbol) → dict
  • get_news_sentiment(symbol) → float # -1 to +1
Computed Indicators
RSI-14 BB(20, 2σ) VWAP deviation ADX-14 ATR-14 MA20 / MA50 News sentiment

Signal Scoring System
A trade requires at least 3 points out of a possible 6 to enter
Minimum entry threshold: 3 points. This gates out weak setups and ensures multiple confirming indicators agree before capital is deployed. The threshold is itself a tunable parameter — the InStrategyAnalyst may recommend raising it to 4 during volatile regimes.
+2
RSI < 32Strong oversold — highest-weight signal
+1
RSI < 40Mild oversold — early stress indicator
+1
Below Lower Bollinger BandPrice outside 2σ — statistically oversold
+1
Below VWAP by > 1%Divergence from intraday volume-weighted price
+1
Positive News SentimentAlpha Vantage sentiment score > 0
+1
ADX < 20 (Ranging)Market regime confirmed non-trending
Trend Filter (HARD BLOCK): If ADX > 25, mean reversion signals are entirely skipped — the trend filter overrides all signal scoring. Mean reversion in a trending market is the primary cause of strategy losses.
🛡️
Hard Rules — Never Violate
Encoded in CLAUDE.md and enforced by OrderExecutor. These cannot be overridden by Claude's reasoning.
1
Position Size ≤ 1%Never risk more than 1% of account value per trade. Position sizing is calculated at order time.
2
Daily Loss Limit: −2%If day P&L hits −2% of portfolio, stop all trading. The bot shuts down until the next trading day.
3
Max 3 Open PositionsNever hold more than 3 simultaneous positions. Enforced by OrderExecutor before order submission.
4
Stop Loss Always SetEvery order must include a stop loss at order time. No order without a stop — ever.
5
Time Gate: No new entries after 14:55 ESTNo overnight holds on equities. All positions closed before market close.
6
Trend Filter: ADX > 25 → SkipIf ADX > 25 on any symbol, do NOT take mean reversion signals on that symbol.
7
Signal Threshold: 3 of 6Require at least 3 signal points before entering. Single-indicator setups are noise, not edge.
8
No Leverage — Spot OnlyNever trade on margin. Cash-funded positions only. Leverage multiplies losses in mean reversion drawdowns.
9
News Filter: Skip Earnings ±48hIf a symbol has earnings announced within 48 hours, skip it. Earnings crush mean reversion edge.
10
Paper First — AlwaysDefault is paper trading. Two env flags must both be explicitly set to enable live trading.

Exit Rules
How and when positions are closed — partial, hard stop, time stop
🎯

Take Profit (Mean Reversion Target)

Exit target is the 20-day simple moving average — "the mean" the strategy is reverting to.

  • Partial exit at mean: Take 50% of the position off when price hits MA20
  • Trail remainder: Move stop to 0.25x ATR trailing stop on remaining 50%
  • Goal: Lock in the core gain while letting the winner run slightly
⛔

Stop Loss Rules

Two stop loss mechanisms protect capital:

  • Hard stop: Set at entry − 0.5 × ATR at order time — never moved down
  • Time stop: Exit any position open for more than 3 days regardless of P&L
  • ATR calibration: Stop width is proportional to the symbol's recent volatility

Journal System
Persistent JSONL logging — every cycle produces an entry, trade or not

trades.jsonl — Trade Journal

Append-only. One JSON record per trading cycle. Never overwritten.

  • timestampISO 8601 EST
  • actionBUY / SELL / HOLD / SKIP
  • symbole.g. "NVDA"
  • signal_score0–6 integer
  • signals_fired["RSI<32", "BelowBB", ...]
  • entry_price / stop_loss / take_profitfloats
  • execution_statusSUBMITTED / REJECTED / SKIPPED
  • reasoningClaude's plain-language explanation
  • account_value / day_plat time of decision

suggestions_in / suggestions_out.jsonl

One record per suggestion. Upserted by category — pending suggestions are updated in-place; actioned ones are kept forever.

  • idin-YYYYMMDD-NNN or out-...
  • categoryrsi_threshold, adx_filter, ...
  • statuspending / approved / archived / cancelled
  • titleshort description
  • analysisfull reasoning text
  • current_rule / proposed_rulebefore/after strings
  • proposed_claude_md_diffexact CLAUDE.md patch
  • confidenceHIGH / MEDIUM / LOW
  • actioned_atISO timestamp when actioned
