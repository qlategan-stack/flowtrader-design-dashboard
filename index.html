System Overview
What FlowTrader is and how it works at a glance
🎯

Strategy

CLAUDE.md

Mean reversion — buys when an asset is statistically oversold relative to its short-term average, exits when price returns to the mean.

  • Indicators: Bollinger Bands, RSI, VWAP, ADX
  • Signal gate: 3 of 6 signals must fire
  • Target exit: 20-day moving average
🌍

Universe

config.yaml

Trades US equities and ETFs via Alpaca. Crypto support via CCXT (Binance) is in the config but not the primary focus.

  • Equities: NVDA, AAPL, MSFT, QQQ, SPY, AMD, GLD
  • Crypto: BTC/USDT, ETH/USDT, SOL/USDT
  • Broker: Alpaca (paper & live)
⚙️

Execution Mode

.env / secrets

Paper trading is the default and requires explicit dual-flag override to go live. Capital safety is the top priority.

  • Default: Paper trading on Alpaca sandbox
  • Live gate: Two env flags must both be set
  • Notifications: Telegram alerts on every trade

TradingDecisionAgent

agents/decision.py · claude-sonnet-4-6
Purpose

The brain of the system. Receives a full market snapshot (prices, indicators, news sentiment, account status) and uses the Claude claude-sonnet-4-6 model to output a structured JSON trade decision. The system prompt is loaded from CLAUDE.md, which encodes all hard rules and signal scoring definitions. Also loads the weekly research memo from journal/weekly_research_memo.json and injects a research context into every prompt — including regime, confidence, flagged opportunities, earnings avoidances, and key risk warnings.

Key Methods
  • analyze_market(market_snapshot, account_snapshot) → dict
  • analyze_single_symbol(symbol_data, account) → dict
  • run_weekly_review(entries) → str
  • _load_research_memo() → dict # reads journal/weekly_research_memo.json
  • _build_research_context() → str # injected into every prompt
Output Format
BUY / SELL / HOLD / SKIP signal_score 0-6 entry + stop_loss + take_profit reasoning + journal_entry

OrderExecutor

agents/executor.py · Alpaca Trading API
Purpose

Validates every trade decision against hard guardrails before submitting to Alpaca. Acts as a final safety layer — even if Claude returns a BUY, the executor will reject if any guardrail fails. Defaults to paper trading; live requires two env flags.

Key Methods
  • validate_order(symbol, side, qty, entry, stop, acct_val, positions, day_pl) → (bool, str)
  • calculate_quantity(account_value, entry_price, stop_loss, risk_pct=0.01) → int
  • place_order(decision, account) → dict
Guardrails Enforced
Max 3 positions 2% daily loss limit 10% max position size Min $100 order value Stop loss required Stop ≤ 5% from entry Paper default

ResearchAnalyst

agents/researcher.py · claude-sonnet-4-6 + Alpha Vantage
Purpose

Weekly market strategist that runs every Sunday evening. Scans a 30-symbol broader universe for mean reversion candidates, fetches VIX, all 11 sector ETF performances, the earnings calendar, and Alpaca market news. Sends everything to Claude to produce a structured JSON weekly memo with watchlist recommendations, sector focus, risk warnings, and a trading confidence score (1-10) for the coming week. The memo is stored at journal/weekly_research_memo.json and displayed in the dashboard's Research tab.

Key Methods
  • run_full_analysis(current_watchlist) → dict # summary
  • get_vix_level() → dict # VIX + risk_level
  • get_sector_performance() → list # 11 sectors ranked
  • get_broader_universe_scan() → list # top 10 candidates
  • get_earnings_calendar() → list # 7-day lookahead
  • get_market_news_summary() → list # Alpaca news headlines
  • generate_weekly_memo(...) → dict # Claude's structured memo
  • save_memo(memo) → (json_path, md_path)
  • load_current_memo() → Optional[dict] # checks expiry

Weekly Research Memo

journal/weekly_research_memo.json · journal/weekly_research_memo.md
Output Structure

The memo is a structured JSON file written every Sunday. The dashboard's Research tab reads it and renders each section. It expires after 7 days and shows a warning if stale. The TradingDecisionAgent also reads it at runtime and injects a research context block into every Claude prompt.

  • market_regimeTRENDING / RANGING / UNKNOWN (or rich dict)
  • confidence_score1–10 integer
  • confidence_reasonone-line rationale
  • top_opportunitieslist of {symbol, reason, signal_strength}
  • watchlist_changes{add, remove, avoid_earnings}
  • sector_focus{favour, avoid}
  • risk_warningslist of {severity, warning}
  • generated_at / valid_untilISO timestamps
  • raw_analysisfull Claude response text
Notification
Telegram summary on completion Markdown copy at weekly_research_memo.md

InStrategyAnalyst

agents/analyst_in.py · daily at 17:30 EST
Purpose

Daily end-of-day analyst that reviews recent trade journal entries and proposes improvements within the existing mean reversion strategy — parameter tuning, signal threshold adjustments, symbol-level observations. Writes suggestions to journal/suggestions_in.jsonl.

Key Methods
  • run(days=30) → list[str] # suggestion IDs written
Output
journal/suggestions_in.jsonl Telegram notification on completion

OutStrategyAnalyst

agents/analyst_out.py · daily at 17:30 EST
Purpose

Complementary daily analyst that looks beyond the current strategy — identifies new potential strategies, market opportunities, or structural improvements that may warrant adding a second strategy layer or expanding the trading universe. Writes suggestions to journal/suggestions_out.jsonl.

Key Methods
